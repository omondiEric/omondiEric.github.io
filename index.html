Firestone

This was a semester-long group project aimed at implementing an emulation of the popular game Hearthstone, built in Clojure. Through this project, not only was I able to delve into the world of functional programming, I was also able to think about how unintended complexity can be greatly reduced by focussing on pure functions. Development was carried out in an agile manner, with bi-weekly sprints, to try and mimic the experience of a real-world software development life cycle. Main topic covered included:

  • Good code design and creating appropriate abstractions to avoid complexity as the code base grows.
  • Data consistency and use of persistent data structures.
  • Designing for testability; Unit tests for easier debugging and understanding.
  • State management and concurrency.

FAT File System

The purpose of this project was to explore storage technologies, specifically focussing on file systems. Using C, we developed a FAT-like file system to support the creation, deletion and modification of files and directories, keeping the general structure similar to the Microsoft FAT design. Some of the main topics explored included memory management and debuggung. project 

Unix Utilities

Implemented "lite" versoins of some important Unix utility functions in relation to the storage and access of data. These included cat, grep, zip and unzip. Through this project, I gained familiarity with unix system calls, how to resourcefully utilize the Linux man pages, Unix file systems, and debugging with GDB. project 

Where You Art

Semester-long group project aimed at designing an application that will aid museum visitors to explore past installations in real time. It covered all phases of the user experience design lifecycle, including refining  design concepts by gathering user data and iterating on interactive prototypes. I worked with two other team members to design a mobile application for museums, that would enable visitors to access more information about various installations based on which part of the museum they are in. Users would also be able to obtain information on past installations that are not currently on display.
